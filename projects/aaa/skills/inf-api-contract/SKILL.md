# inf-api-contract

> Project: aaa. Used by RS-Builder + RS-Reviewer. Repos: `infina-insurance-partner-services` (BE) ↔
> `infina-insurance-partner-webapp` (FE). The FE consumes BE types generated by openapi-typescript.

## When to use
Any change that touches the BE HTTP/OpenAPI surface (controllers, DTOs, response shapes) OR any FE
code that reads `libs/types/src/api.gen.ts`.

## The contract
- BE exposes an OpenAPI spec. FE runs `yarn gen:api` (openapi-typescript) to regenerate
  `libs/types/src/api.gen.ts` (+ related `*.gen.ts`). FE consumers MUST match the regenerated types.
- `gen:api` needs the BE reachable: `USER_API_URL` / `ADMIN_API_URL` (default `http://localhost:3333`).

## Builder steps (when you changed the BE surface)
1. Start the local BE: `yarn start` (→ :3333) — the worktree's copied `.env` auto-loads.
2. Regenerate FE types: `USER_API_URL=http://localhost:3333 ADMIN_API_URL=http://localhost:3333 yarn gen:api`.
3. Fix every FE consumer the new types break; `nx typecheck` must be clean.
4. **Commit the regenerated `libs/types/*.gen.ts`** in the same branch. Never hand-edit generated files.
5. In the hand-off note, **explicitly call out the contract change** + which FE consumers were updated.

## Reviewer steps (3-lens CONTRACT lane)
1. If the diff changes the BE OpenAPI surface or `*.gen.ts`, confirm `yarn gen:api` was re-run and the
   regenerated files are committed (not hand-edited, not stale).
2. **⚠️ MANDATORY-HUMAN-REVIEW flag:** any `api.gen.ts` / contract change is the reviewer's known weak
   spot — mark it, @mention the owner, and **do not approve it alone**.
3. Verify FE consumers compile against the new types and no consumer was missed.

## Done when
BE surface + `*.gen.ts` are consistent, `nx typecheck` is clean across affected FE apps, the change is
called out, and any contract change is escalated for human review.
